## Combat text system

ONE node total: 'CombatText' (Node2D in the run scene, z_index ~90, world-space canvas). It owns a PREALLOCATED fixed pool of 16 entry slots (plain dicts created once at boot: {pos, age, text, color, size, priority, alive}) and renders every live entry in a SINGLE _draw pass — draw_string_outline (1-2px C1 void #0A001A) then draw_string with a cached font (existing pixel/UI font). No Label nodes, no tweens, no per-proc allocation ever: push() writes into a dead slot or evicts by priority. One _process loop mutates age and issues queue_redraw — animation per entry: pop-in scale 1.15→1.0 over 0.1s, rise ~28px over 0.6s with ±6px x-jitter, fade over the last 0.25s. API: CombatText.crit(world_pos, amount) and CombatText.callout(world_pos, word, color).

RULES: (a) NUMBERS = crit damage ONLY — gold, roundi(damage), 18px, spawned from the same ctx['crit'] plumbing at the 5 hit sites; per-enemy 0.15s ICD so beam/cone crits can't stack a gold column. Double Tap's echo number rides the same path. (b) CALLOUTS = headline-proc words only — family color, 22px. Approved vocabulary (10): FRENZY (orange), SURGE (lavender), SHATTER (indigo/white), NOVA (orange, t3 only), RUPTURE (green), OUTBREAK (green), EXECUTED (red), FIRED. (red, Pink Slip), BLACK FRIDAY (indigo), GRAVEYARD SHIFT (red, arm transition only). Buff callouts fire only on the inactive→active transition. (c) DEDUPE: an identical callout word within 0.5s REFRESHES the existing entry (reset age, small re-pop) instead of taking a slot — a horde of shatters reads as one pulsing word, not a wall. (d) CAPS: 16 concurrent entries, max 4 new pops per frame (excess silently dropped, never queued); on pool exhaustion steal the oldest lowest-priority entry (callouts outrank crit numbers). (e) NO per-hit damage numbers, no heal text, no filler — small procs are tints/motes/rings only, per Larry's spec (Brass Picker's '+1' lives as a HUD ammo-counter blip, not floating text). Worst case is 16 strings in one canvas item — trivial on phone.

## Risks (load-bearing constraints)

(1) TEXT PERF — must be the single-canvas-item pooled _draw design; a Label per proc tanks horde frames. Caps are load-bearing: 16 slots, 4 pops/frame, per-enemy crit ICD 0.15s, 0.5s callout dedupe. (2) VFX FLOOD — 21 guns x multishot x 60 enemies: chain + Death Rattle + explode rings + shatters can spike transient nodes. The TalentEngine per-frame VFX budget (~6 spawns/frame, gameplay unaffected) ships in Phase 1, not after. (3) POOL CAP SHARING — Bile Spill MUST ride the same MAX_PLAYER_POOLS eviction as the Acid Cannon (lift Bullet._cap_player_pools to a shared static) or floors carpet with 5 Hz-tick zones. (4) 5-SITE PLUMBING — ctx['crit'] (crit numbers + Double Tap) touches Bullet, Gun cone/beam/lightning, and Shockwave.blast; miss one site and that weapon class silently never crits visibly (same trap class as the LocalHudBuilder bug) — add a per-site checklist + headless probe. (5) SAME-FRAME CORPSE READS — Outbreak's status_snapshot and Death Rattle's corpse position read a queue_free'd node: valid same-frame, garbage if ever deferred — snapshot immediately, never call_deferred. (6) RECURSION CONTAINMENT — every new damage source (echo, mines, pools, bolts, novas, ruptures) deals raw take_damage / carries no payload (Dead Man's Switch passes gun=null); detonate() stays proc-free. Enforce in review — one mine proccing Gut Bomb proccing mines is an infinite loop. (7) TINT LADDER — fear adds a state and burn/poison/mark/slow become blends; keep ONE fixed priority in the static _resolve_tint and update the headless precedence probe (its signature changes). Pulses/blends, never solid washes (FLASH_CD lesson). (8) PRE-EXISTING OVERWRITE BUG — resolve_payload's reload_nova and overpen single keys silently drop one talent when kinds repeat across tiers (now guaranteed: Short Fuse+Backblast+Powder Keg, Rebar+Railbreaker). Merge/sum in the branch — ship WITH the data phase. (9) BALANCE HOTSPOTS — Curb Stomp + Closing Time makes 'hampered' near-permanent around the player (~+30% global); acceptable as the CC payoff but don't let cc_bonus stack across copies. Executes now at 4 rows (Pink Slip t1 threshold must stay tiny, 3-6%). Dead Man's Switch's 6s ICD and Black Friday's cap-1 + no-damage are the levers. Frenzy is max-wins across 4 sources — document, don't 'fix'. (10) FEAR FEEL — feared runners dragging ranged enemies off-screen; cap <= 2s, boss-immune via has_method exactly like pin. (11) KNOCKBACK INTERPLAY — Black Friday pulls share the knockback channel with bite-and-bounce and Concussive; the 900/s decay resolves them, but verify the bounce-out still wins point-blank so contact damage doesn't double-tick. (12) POOL DILUTION — t1 grows 5→15: new pulls feel different (Killshot odds 1/5→1/15) — intended rebalance, but expect Larry to notice; note in changelog. LootRoller dedupe is safe (t3 pool 23 >> 5 Armageddon slots); rarity counts + slot ladder untouched — pure catalog addition. (13) MOD ROLL DIRECTION — the roll scheme assumes max=better on EVERY mod; never add interval-style mods (all rates here are per-second or chance-%). (14) SCOPE HONESTY — 16 NEW kinds vs ~14.5 target, but 8 of them are <=6-line branches reusing shipped channels (pin/ammo/cc_bonus/bolt/lowhp-frenzy/aura-slow/first-shot/last-call) and only TWO bespoke nodes exist in the whole roster (Mine, GravityWell) — implementation weight lands at ~50/50. Flagged rather than padded with stat clones.
