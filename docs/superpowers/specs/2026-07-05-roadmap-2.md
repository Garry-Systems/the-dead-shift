## Mini-specs (locked decisions; implementers read code first; numbers = starters in GameConfig)

### A. Run variety (v0.1.50)
**Elites** — modifier system on existing trash (bosses excluded): spawn roll per spawn, `ELITE_CHANCE_BASE 0.05 + 0.005/wave, cap 0.15, min_wave 6`. Elite = ONE modifier + ×2.5 HP + gem value ×3 + `RunStats.add_coins(ELITE_COIN_BONUS 5)` on kill. Modifiers (colored outline ring per family, drawn in Enemy._draw over the base shape): **Armored** (C3 metallic ring, takes −30% damage — applied in take_damage pre-guard), **Volatile** (green ring, on death a telegraphed 0.6s fuse then a player-hurting blast = exploder constants ×1.2 — reuse that flow), **Splitter** (C4 ring, on death spawns 2 runners at 50% of ITS hp, no re-split, no elite inheritance), **Alpha** (gold ring, aura: enemies within 300px get +20% speed/damage — 5Hz refresh like AuraRing, buff clears when Alpha dies). Wired via Spawner post-configure (roll + apply), fields via .get defaults so raw spawns unaffected.
**Night events** — `NightEvents` node in Main (endless only): at each wave start past wave 4, `EVENT_CHANCE 0.22` if none active; duration ~1 wave. Announce via the Hud banner + dawn-sting audio; end with a short "ALL CLEAR" callout. Events: **BLOOD MOON** (spawn interval ×0.5, `RunStats.coin_mult +1.0` while active — restore after, compose with the card multiplier by ADDING then removing the same amount; red CanvasModulate tint ~15%), **FOG BANK** (CanvasModulate dim ~35% + gem value ×2 while active), **POWER SURGE** (player electric: +2 chain jumps on gun lightning AND talent chains while active — a GameConfig-read bonus consumed where jumps resolve), **RUSH HOUR** (warning banner then 6-10 extra car/rubble cover scattered in a rough corridor near the player via ObstacleField's placement helpers — keep-out + caps respected; permanent normal obstacles afterward, cullable). All magnitudes/durations = consts.
**Dawn extraction** — endless only. At 6:00 AM (existing dawn hook): banner becomes "RESCUE INBOUND — SURVIVE TO EXTRACT", `FINAL_SURGE_SECONDS 90` of forced spawn-floor + elite chance ×2; then a code-drawn chopper (palette, rotor spin _draw) descends to an LZ marker circle near the forecourt; player has `EXTRACT_WINDOW 20`s to stand in the LZ → **WIN**: pay stub reframes "SHIFT SURVIVED — CLOCKED OUT ALIVE", coins ×`EXTRACT_PAY_MULT 1.5` on the total, confetti, new save stat `shifts_survived`, then back to menu. Ignore the chopper → it leaves, endless continues (the current dawn bonus already paid as-is). Boss rush untouched. The existing DAWN_BONUS_COINS stays and pays at 6 AM regardless.

### D. Stats + juice (v0.1.51)
**Records** — SaveManager lifetime counters (schema-extend, updated at payout time from RunStats: kills, runs, coins_earned, bosses, elites, best clock-out string, shifts_survived, armageddons_pulled [increment in LootRoller/Inventory add when rarity==9], per-gun kill dict capped to base ids). MainMenu RECORDS view (button on hub, PixelTheme list, drag-scroll like store).
**Juice** — crit-KILL hit-stop: `Engine.time_scale 0.05` for 0.05s real-time then restore (guard re-entry, skip if already stopped; consts); screen shake: camera offset noise, magnitude by source (blast radius scaled, boss slam, player hurt), cap + `SHAKE_ENABLED` save toggle in pause next to sound toggles; extraction chopper + dawn get the big shake. No music reactivity (skip).

### B. Weapon fusion (v0.1.52)
In WeaponDetailPopup: **FEED** button → picker of owned same-BASE duplicates (not equipped, not the viewed gun) → confirm → consume sacrifice, grant weapon XP = sacrifice's scrap midpoint × `FUSION_XP_MULT 2.0` (uses existing add-XP/level path so talents unlock live); if the sacrifice was SAME RARITY (or higher), additionally reroll the viewed gun's LOWEST-quality stat roll to a fresh `snappedf(randf(),0.001)` with a "REROLLED: <stat>" line in the result. Popup shows XP gained + any unlocks. Guards: equipped can't be fed as sacrifice (existing deconstruct guard pattern), crates unaffected. Pure Inventory/WeaponInstance/UI — no save migration (instances already store xp/level/stats).

### C. Challenges + daily (v0.1.53)
**Challenge board** — data pool `Challenges.gd` (~12 rows: kill N with fire/electric/poison talents, kill N elites, reach 2:00 AM, survive a Blood Moon, no-dash to midnight, open N crates, etc.; each: id, desc template, target, reward crate id). 3 active, rotated daily (same date mechanism as the daily reward; completed ones replaced next day). Progress: RunStats counters flushed to save at payout; completion popup (RewardPopup variant) grants the crate. Surface: a CHALLENGES section on the hub or records view.
**Daily shift** — one attempt/day (save-gated like the daily claim): DAILY SHIFT button on mode select → seeds a run-scoped RNG (`RandomNumberGenerator` seeded from the date string hash) used for NightEvents rolls + elite rolls + spawn-type picks (loot/talents stay global RNG — full determinism NOT attempted, document); pay stub gains "DAILY SHIFT — <date>" header + your score; best daily score saved. No servers.

### E. Characters (v0.1.54)
**The Janitor** (2800c): dash leaves a mop-bucket slick (HazardZone: hurts nobody, `slow 0.5/2s` on enemies, C4-tinted wet-floor look, pool-cap shared); passive: +1 coin per kill (RunStats bonus_coins). **The Delivery Girl** (3200c): dash drops an armed Parting-Gift mine (reuse Mine, same caps); passive: +20% pickup radius. Both auto-wire via Characters.all() (store/select). Dash abilities via the existing dash_ability match pattern.

### F. Sprites (v0.1.55)
Home-repo generator (`gen_palette_sprites.py` new `enemies()` + `bosses()`): 32×32 strict-palette sprites for shambler/runner/brute/exploder/hive/spitter/mutant (distinct silhouettes, C3 bodies shaded like the gunmetal ramp but palette-strict) + 48×48 bosses (Brute/BroodMother/HeatTyrant/Manager/NightStocker/Fryer/Courier w/ their staff props). Game side: Enemy/BossBase render Sprite2D textures instead of _draw shapes; hit-flash/tint shader must keep working (verify how the flash shader binds — Sprite2D modulate vs shader param); elite outline rings still draw OVER the sprite. Multimodal contact-sheet QA (controller reviews the sheet before ship, like the gunmetal set). Fallback: any enemy without a sprite keeps its _draw shape (staged rollout safe).
